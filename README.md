# ai-agent-template

Claude Code（AIエージェント）を使って業務を遂行するための、非エンジニア向けリポジトリテンプレートです。

営業企画・業務設計・PdM などの職種を問わず、**fork してすぐに使える**設計になっています。

---

## このテンプレートでできること

- AIエージェントに業務タスクを依頼し、品質の高い成果物を得る
- 成果物の品質ゲートを自動チェックする
- 業務での失敗パターンを蓄積・学習して自己進化する
- 職種・業務に合わせてエージェントをカスタマイズする

---

## クイックスタート

### 1. このリポジトリを fork する

右上の「Fork」ボタンをクリックして、自分のアカウントにコピーします。

### 2. 不要なロールエージェントを削除する

自分の職種に関係ないロールのフォルダを削除します。

```
agents/sales-planning/    ← 営業企画以外の人は削除
agents/business-design/   ← 業務設計以外の人は削除
agents/pdm/               ← PdM以外の人は削除
```

### 3. Claude Code でこのリポジトリを開く

Claude Code（[claude.ai/code](https://claude.ai/code) または VSCode拡張）でフォルダを開きます。

### 4. タスクを依頼する

例えばこう話しかけます：

```
競合A社と自社の比較提案書を作りたい。
対象顧客は中規模の小売業で、課題は在庫管理の非効率化。
```

Claude が計画・調査・執筆・レビューの流れで進めます。

## ディレクトリ構成

```
ai-agent-template/
├── CLAUDE.md              # AI OS のコア設定（エージェントが最初に読む）
├── AGENTS_COMMON.md       # 全エージェント共通ルール
├── SKILL_MODEL_MAP.json   # タスク別の推奨モデル
├── HOW_TO_ADD_ROLE.md     # ロール追加・削除の方法
├── feature_list.json      # タスク管理リスト
│
├── agents/
│   ├── _researcher/       # 調査エージェント（共通）
│   ├── _writer/           # 文書作成エージェント（共通）
│   ├── _reviewer/         # レビューエージェント（共通）
│   ├── sales-planning/    # 営業企画ロール（不要なら削除）
│   ├── business-design/   # 業務設計ロール（不要なら削除）
│   └── pdm/               # PdMロール（不要なら削除）
│
├── memories/
│   ├── lessons_learned/   # 失敗・学習の記録（自動蓄積）
│   ├── summaries/         # 蒸留済みアーカイブ
│   └── confirmation_history.md  # 確認履歴
│
├── plans/                 # タスクの作業計画
├── results/               # タスクの完了記録
├── outputs/               # 完成した成果物
├── scratchpad/            # 下書き・試行錯誤（最終成果物に含めない）
└── templates/             # よく使う文書テンプレート
    ├── proposal.md        # 提案書
    ├── meeting-minutes.md # 議事録
    ├── requirements.md    # 要件定義書
    └── roadmap.md         # ロードマップ
```

## 自分のロールを追加する

`HOW_TO_ADD_ROLE.md` を参照してください。人事・財務・マーケ・法務など、どんな職種でも追加できます。

---

## 設計思想

このテンプレートはシステム開発のAIエージェント設計を、非エンジニアの業務遂行向けに再設計したものです。

- **計画先行**: 全タスクを `plans/` に記録してから実行する
- **品質ゲート**: 完成物が基準を満たすまで `outputs/` に入れない
- **自己進化**: 失敗パターンを蓄積し、CLAUDE.md に昇格させる
- **モデル最適化**: タスクの重要度に応じて適切なモデルを提案する
