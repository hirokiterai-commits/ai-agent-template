# AGENTS_COMMON.md — 全エージェント共通ルール v1.0
> **CLAUDE.md の次に全エージェントが読むファイル。更新権限: 人間のみ。**
> 最終更新: 2026-05-11

---

## 1. パイプライン全体図

```
人間
 ↕（タスク指示・承認・判断依頼）
Claude（Orchestrator役）
 ↓
Researcher → Writer → Reviewer → 人間承認 → outputs/
               ↑          ↓FAIL
               └──────────┘（差し戻し・最大2回）
```

> タスクの性質によって不要なステップはスキップしてよい。
> 例: 調査不要の定型文作成 → Writer のみ。

---

## 2. モデル選択

各セッション開始前に `SKILL_MODEL_MAP.json` を参照し、タスクの性質から適切なモデルを人間に提案する。

- **コストが気になる場合**: 「このタスクは〇〇スキルなので Haiku で十分です。`/model` で切り替えますか？」と提案する
- **品質が重要な場合**: 「この分析は重要度が高いため Opus を推奨します」と提案する
- 人間が判断・操作する。Claude は提案するだけで強制しない。

---

## 3. コンテキスト引き継ぎの原則

後続エージェントには **判断に必要な最小情報のみ** を渡す。

| 引き継ぎ元 → 先 | 渡すもの | 渡さないもの |
|---|---|---|
| Researcher → Writer | 調査サマリー・出典リスト・要確認フラグ付き数値 | 調査の生データ全体 |
| Writer → Reviewer | 成果物・想定読者・目的一文 | 下書き過程 |
| Reviewer → 人間 | レビュー結果（良い点・要修正・提案） | レビューの思考過程 |

---

## 4. タスクID の採番ルール

```
TASK_ID = YYYYMMDD-{3桁連番}
例: 20260511-001
```

---

## 5. 差し戻しルール

`results/{タスクID}-draft.json` の `rework_count` フィールドで管理する。

```
rework_count < 2  → 通常の差し戻し
rework_count >= 2 → 人間にエスカレーション（方針確認を求める）
```

---

## 6. 禁止事項

- `AGENTS_COMMON.md` の自己編集
- `CLAUDE.md` の大幅変更・削除（失敗パターン早見表への蒸留追記のみ許可）
- 人間の承認なしの外部送信・配布・公開
- 機密情報・個人情報の `outputs/` への直接記載
- AI 生成の数値・統計を確認なしに成果物に含めること
- `scratchpad/` 以外への下書き保存

---

## 7. 条件付き自己進化（人間承認が必要な操作）

以下は AI が自律的に実施してよい。

| 許可操作 | 条件 |
|---|---|
| `CLAUDE.md` 失敗パターン早見表への追記 | 既存行の削除・変更なし |
| `agents/[ロール]/.clauderules` への本能ルール追記 | 既存行の削除・変更なし |
| `memories/lessons_learned/` への新規エントリ作成 | 既存ファイルの変更は最小限 |
| `memories/confirmation_history.md` の更新 | 定義されたフォーマットに従う |
