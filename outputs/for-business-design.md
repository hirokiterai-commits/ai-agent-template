# 業務設計・オペレーション担当のためのAI Business OS
> 読了時間: 3分

---

## 「AIはなんでもできる」は本当か？

ChatGPT や Claude に聞けばなんでも答えてくれる。そう感じている人は多いはずです。

でも実際に業務で使おうとすると、こんな問題に気づきます。

| 体験した問題 | AIの構造的な弱点 |
|---|---|
| 先週話したことを覚えていない | **記憶がない**（毎回ゼロから説明が必要） |
| 自信満々に間違った情報を出してくる | **事実を作る**（ハルシネーション） |
| 同じ頼み方をしても毎回品質がバラつく | **一貫性がない** |
| チームで使うと人によって全然違う結果になる | **個人のスキル依存** |
| 同じミスを何度でも繰り返す | **学習しない** |

これらは「プロンプトの書き方」の問題ではありません。**AIという技術の構造的な限界**です。

---

## Skillsを作っても解決できない理由

「じゃあ便利なスラッシュコマンドを作ればいい」という発想があります。でもSkillsは**単発の命令を実行するだけ**なので、構造的な問題は何も変わりません。

- 使い忘れたらゼロ
- 記憶は依然としてない
- チームで品質を揃えられない
- 失敗が蓄積されない

**構造的な問題は、構造で解決するしかない。**

---

## このテンプレートが解決する方法

役割を分けて、ルールをファイルに持ち、失敗を学習する仕組みを構造として組み込みます。

| 弱点 | このテンプレートの解決策 |
|---|---|
| 記憶がない | `memories/` にファイルとして永続化。次のセッションでも覚えている |
| 事実を作る | 人間承認フロー必須。「要確認」フラグで数値・固有名詞を区別 |
| 一貫性がない | `.clauderules` に業務設計の判断基準をファイルで固定 |
| 個人のスキル依存 | リポジトリをForkするだけでチーム全員が同じルールで動く |
| 学習しない | 失敗パターンを蓄積し、次回から自動的に回避する |

## 業務設計の現場でいうと

```
「受発注フローをAs-Is/To-Beで整理して。
　影響する部署と反対意見も想定して」
　→ フロー図・比較表・懸念点への回答案が出てくる
　→ 次回は「前回のフロー整理を踏まえて」と言えば覚えている

「新入社員向けの経費精算マニュアルを作って」
　→ 番号付き手順・例外処理込みで出てくる
　→ 品質チェックを通過したものだけが成果物になる
　→ 「前回このマニュアルでこの点が抜けていた」という学習が次回に活きる
```

**普通にClaudeに聞くとの違い：覚えていて、品質が一定で、学習する。**

## 頼める作業の例

| 作業 | 具体的な頼み方 |
|---|---|
| 業務フロー整理 | 「現在の請求処理フローをAs-Is/To-Beで整理して。改善ポイントと難易度も添えて」 |
| マニュアル作成 | 「新入社員向けの勤怠申請マニュアルを作って。例外処理も含めて」 |
| 改善提案 | 「月次レポート作成の工数を削減したい。現状の課題と改善案を出して」 |
| ステークホルダー分析 | 「このフロー変更に反対しそうな部署と、その懸念への回答案を作って」 |
| 要件整理 | 「新システム導入の要件を整理して。必須・あれば良い・対象外に分けて」 |
| WBS作成 | 「このプロジェクトのWBSを作って。担当・期限・依存関係も含めて」 |

## 使い始め方

**所要時間: 30分以内**

### Step 1: GitHubでForkする
`github.com/hirokiterai-commits/ai-agent-template` を開いて右上の「Fork」ボタンを押す。自分のGitHubアカウントにコピーされます。

### Step 2: GitHub Desktopでcloneする
1. [GitHub Desktop](https://desktop.github.com/) をインストール
2. GitHubアカウントでサインイン
3. `File → Clone repository` を開く
4. 「GitHub.com」タブから自分のForkしたリポジトリを選択して「Clone」

これで自分のPCにダウンロードされます。

### Step 3: Claude Codeで開く
Claude Code（VSCode拡張 or `claude.ai/code`）でcloneしたフォルダを開く。

### Step 4: 不要なフォルダを削除する
「業務設計担当として使いたい。どのフォルダを削除すればいい？」と聞く。指示通りに削除して完了。

あとは日本語で話しかけるだけです。

## よくある疑問

**Q. 社内の機密情報を入力して大丈夫？**
A. 実名・社外秘データはそのまま入力しないでください。「A社」「B部門」のように仮名化してから渡してください。

**Q. JIRAと連携できる？**
A. できます。タスクの進捗管理をJIRAと連動させることができます。セットアップ手順は `JIRA_SETUP.md` を参照してください。
