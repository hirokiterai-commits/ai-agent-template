# デザイナーのためのAI Business OS
> 読了時間: 3分

---

## 「AIはなんでもできる」は本当か？

ChatGPT や Claude に聞けばなんでも答えてくれる。そう感じている人は多いはずです。

でも実際に業務で使おうとすると、こんな問題に気づきます。

| 体験した問題 | AIの構造的な弱点 |
|---|---|
| 先週話したことを覚えていない | **記憶がない**（毎回ゼロから説明が必要） |
| 自信満々に間違った情報を出してくる | **事実を作る**（ハルシネーション） |
| 同じ頼み方をしても毎回品質がバラつく | **一貫性がない** |
| チームで使うと人によって全然違う結果になる | **個人のスキル依存** |
| 同じミスを何度でも繰り返す | **学習しない** |

これらは「プロンプトの書き方」の問題ではありません。**AIという技術の構造的な限界**です。

---

## Skillsを作っても解決できない理由

「じゃあ便利なスラッシュコマンドを作ればいい」という発想があります。でもSkillsは**単発の命令を実行するだけ**なので、構造的な問題は何も変わりません。

- 使い忘れたらゼロ
- 記憶は依然としてない
- チームで品質を揃えられない
- 失敗が蓄積されない

**構造的な問題は、構造で解決するしかない。**

---

## このテンプレートが解決する方法

役割を分けて、ルールをファイルに持ち、失敗を学習する仕組みを構造として組み込みます。

| 弱点 | このテンプレートの解決策 |
|---|---|
| 記憶がない | `memories/` にファイルとして永続化。次のセッションでも覚えている |
| 事実を作る | 人間承認フロー必須。「要確認」フラグで数値・固有名詞を区別 |
| 一貫性がない | `.clauderules` でデザイナーとしての判断基準をファイルに固定 |
| 個人のスキル依存 | リポジトリをForkするだけでチーム全員が同じルールで動く |
| 学習しない | 失敗パターンを蓄積し、次回から自動的に回避する |

---

## デザイナーの業務でいうと

```
「競合サービスのUIをまとめて。比較表とポイントで」
　→ 出てくる（数分）
　→ 次回も「前回の比較を踏まえて」と言えば覚えている

「このページのコピーを3パターン考えて。ターゲットは20代女性」
　→ 3パターン＋意図の説明が出てくる
　→ 品質チェックを通過したものだけが成果物になる
　→ 「前回このパターンは不評だった」という学習が次回に活きる
```

**普通にClaudeに聞くとの違い：覚えていて、品質が一定で、学習する。**

## 頼める作業の例

| 作業 | 具体的な頼み方 |
|---|---|
| 競合・参考事例の調査 | 「ECサイトのカート離脱対策のUI事例を5つまとめて」 |
| コピーライティング | 「CTAボタンのテキストを3パターン。簡潔・親しみやすい・信頼感の3方向で」 |
| デザインブリーフ作成 | 「新機能のデザインブリーフを作って。目的はユーザーの継続率向上」 |
| 議事録・仕様書 | 「さっきのデザインレビューの内容をまとめて。決定事項と宿題を分けて」 |
| プレゼン資料の構成 | 「デザイン改善提案のスライド構成を作って。経営層向け、5分で話せる量で」 |

## 使い始め方

**所要時間: 30分以内**

### Step 1: GitHubでForkする
`github.com/hirokiterai-commits/ai-agent-template` を開いて右上の「Fork」ボタンを押す。Fork作成画面で **「Private」を選択してください**（業務ルール・設定が含まれるため）。自分のGitHubアカウントにコピーされます。

### Step 2: GitHub Desktopでcloneする
1. [GitHub Desktop](https://desktop.github.com/) をインストール
2. GitHubアカウントでサインイン
3. `File → Clone repository` を開く
4. 「GitHub.com」タブから自分のForkしたリポジトリを選択して「Clone」

これで自分のPCにダウンロードされます。

### Step 3: VS Code と Claude Code をセットアップする
1. [VS Code](https://code.visualstudio.com/) をインストール
2. VS Code を開き、左サイドバーの拡張機能アイコン（四角が4つのアイコン）をクリック
3. 検索欄に「Claude Code」と入力してインストール
4. インストール後、Claudeアカウントでサインイン

### Step 4: cloneしたフォルダを開く
VS Code の `File → Open Folder` でStep 2でcloneしたフォルダを開く。左下にClaude Codeのアイコンが表示されればセットアップ完了です。

### Step 5: 不要なフォルダを削除する
「デザイナーとして使いたい。どのフォルダを削除すればいい？」と聞く。指示通りに削除して完了。

あとは日本語で話しかけるだけです。
