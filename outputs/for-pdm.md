# PdMのためのAI Business OS
> 読了時間: 3分

---

## 「AIはなんでもできる」は本当か？

ChatGPT や Claude に聞けばなんでも答えてくれる。そう感じている人は多いはずです。

でも実際に業務で使おうとすると、こんな問題に気づきます。

| 体験した問題 | AIの構造的な弱点 |
|---|---|
| 先週話した要件を覚えていない | **記憶がない**（毎回ゼロから説明が必要） |
| 技術的に不可能なことを断言してくる | **事実を作る**（ハルシネーション） |
| 同じ頼み方をしても毎回品質がバラつく | **一貫性がない** |
| PdMによって要件定義の粒度・品質が全然違う | **個人のスキル依存** |
| 同じ設計ミスを何度でも繰り返す | **学習しない** |

これらは「プロンプトの書き方」の問題ではありません。**AIという技術の構造的な限界**です。

---

## Skillsを作っても解決できない理由

「じゃあ便利なスラッシュコマンドを作ればいい」という発想があります。でもSkillsは**単発の命令を実行するだけ**なので、構造的な問題は何も変わりません。

- 使い忘れたらゼロ
- 記憶は依然としてない
- チームで品質を揃えられない
- 過去の設計判断が蓄積されない

**構造的な問題は、構造で解決するしかない。**

---

## このテンプレートが解決する方法

役割を分けて、ルールをファイルに持ち、失敗を学習する仕組みを構造として組み込みます。

| 弱点 | このテンプレートの解決策 |
|---|---|
| 記憶がない | `memories/` にファイルとして永続化。過去の要件・設計判断も覚えている |
| 事実を作る | 技術的実現可能性には「要エンジニア確認」フラグを自動付与 |
| 一貫性がない | `.clauderules` にPdMとしての判断基準（KPI必須・Whyの記録など）をファイルで固定 |
| 個人のスキル依存 | リポジトリをForkするだけでチーム全員が同じルールで動く |
| 学習しない | 過去の設計判断・失敗パターンを蓄積し、次回から自動的に活かす |

---

## PdMの現場でいうと

```
「通知機能の要件定義を作って。
　Must/Should/Couldで優先度も整理して」
　→ ユーザーストーリー・KPI・優先度・トレードオフが出てくる
　→ 技術的な断言箇所には「要エンジニア確認」フラグが付く
　→ 次回は「前回の要件定義を踏まえて」と言えば覚えている

「Q3のロードマップを更新して」
　→ 根拠付きのロードマップが出てくる
　→ 「前回この順序で依存関係の問題が起きた」という学習が活きる
```

**普通にClaudeに聞くとの違い：覚えていて、品質基準が一定で、設計判断が蓄積される。**

## 頼める作業の例

| 作業 | 具体的な頼み方 |
|---|---|
| 要件定義 | 「〇〇機能の要件定義を作って。機能要件と非機能要件を分けて、Whyも書いて」 |
| 優先度付け | 「この機能リストをMoSCoWで整理して。根拠も添えて」 |
| ロードマップ | 「Q3〜Q4のロードマップを作って。各フェーズの順序の根拠と依存関係も」 |
| KPI設計 | 「この機能の成功指標を設計して。測定方法と確認タイミングも含めて」 |
| ステークホルダー向け資料 | 「この要件定義を経営層向けに1枚でまとめて。WhyとROIを前面に」 |
| 仕様書 | 「この機能の仕様書を作って。開発チームが実装できる粒度で」 |

## 使い始め方

**所要時間: 30分以内**

### Step 1: GitHubでForkする
`github.com/hirokiterai-commits/ai-agent-template` を開いて右上の「Fork」ボタンを押す。Fork作成画面で **「Private」を選択してください**（業務ルール・設定が含まれるため）。自分のGitHubアカウントにコピーされます。

### Step 2: GitHub Desktopでcloneする
1. [GitHub Desktop](https://desktop.github.com/) をインストール
2. GitHubアカウントでサインイン
3. `File → Clone repository` を開く
4. 「GitHub.com」タブから自分のForkしたリポジトリを選択して「Clone」

これで自分のPCにダウンロードされます。

### Step 3: VS Code と Claude Code をセットアップする
1. [VS Code](https://code.visualstudio.com/) をインストール
2. VS Code を開き、左サイドバーの拡張機能アイコン（四角が4つのアイコン）をクリック
3. 検索欄に「Claude Code」と入力してインストール
4. インストール後、Claudeアカウントでサインイン

### Step 4: cloneしたフォルダを開く
VS Code の `File → Open Folder` でStep 2でcloneしたフォルダを開く。左下にClaude Codeのアイコンが表示されればセットアップ完了です。

### Step 5: 不要なフォルダを削除する
「PdMとして使いたい。どのフォルダを削除すればいい？」と聞く。指示通りに削除して完了。

あとは日本語で話しかけるだけです。

## よくある疑問

**Q. 技術的な実現可能性まで判断してくれる？**
A. しません。技術的な断言箇所には「要エンジニア確認」フラグを自動で付ける設計になっています。最終判断は必ずエンジニアと確認してください。

**Q. JIRAと連携できる？**
A. できます。チケットのステータス管理やロードマップの進捗をJIRAと連動させることができます。セットアップ手順は `JIRA_SETUP.md` を参照してください。
